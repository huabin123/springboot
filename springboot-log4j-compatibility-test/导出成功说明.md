## 🎯 为什么只有3个依赖？

对于 **Spring Boot 2.2.2.RELEASE** 升级 Log4j 到 2.25.3，只需要这3个核心依赖：

### 1. log4j-api (必需)
- Log4j 2 的 API 接口
- 所有日志调用都通过这个 API

### 2. log4j-core (必需)
- Log4j 2 的核心实现
- 负责实际的日志处理、输出等

### 3. log4j-slf4j-impl (必需)
- SLF4J 到 Log4j 2 的桥接
- Spring Boot 使用 SLF4J 作为日志门面，需要这个桥接

### 其他依赖（可选）

以下依赖在基本场景下**不是必需的**：

- `log4j-jul` - 只有使用 JUL (java.util.logging) 时才需要
- `log4j-jcl` - 只有使用 Commons Logging 时才需要
- `log4j-web` - 只有需要 Web 特定功能时才需要
- `log4j-1.2-api` - 只有需要兼容 Log4j 1.x 时才需要

## 🚀 使用方法

### 方式1：导入到内网 Nexus（推荐）

```bash
# 1. 传输压缩包到内网
scp log4j-dependencies-*.tar.gz user@internal-server:/path/

# 2. 在内网解压
tar -xzf log4j-dependencies-*.tar.gz
cd log4j-dependencies-export

# 3. 修改 Nexus 配置
vi import-to-nexus.sh
# 修改：NEXUS_URL、NEXUS_USER、NEXUS_PASSWORD

vi settings.xml
# 修改：username、password

# 4. 执行导入
./import-to-nexus.sh
```

### 方式2：复制到本地仓库

```bash
# 直接复制到本地 Maven 仓库
cp -r log4j-dependencies-export/repository/* ~/.m2/repository/
```

### 方式3：使用本地文件仓库

在项目 `pom.xml` 中添加：

```xml
<repositories>
    <repository>
        <id>local-log4j</id>
        <url>file://${project.basedir}/log4j-dependencies-export/repository</url>
    </repository>
</repositories>
```

## 📝 项目中如何使用

在你的 Spring Boot 2.2.2.RELEASE 项目中：

### 1. 排除默认的 logback

```xml
<dependency>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-web</artifactId>
    <exclusions>
        <exclusion>
            <groupId>org.springframework.boot</groupId>
            <artifactId>spring-boot-starter-logging</artifactId>
        </exclusion>
    </exclusions>
</dependency>
```

### 2. 添加 log4j2 依赖

```xml
<!-- Spring Boot Log4j2 Starter -->
<dependency>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-log4j2</artifactId>
</dependency>

<!-- 强制使用 log4j-api 2.25.3 -->
<dependency>
    <groupId>org.apache.logging.log4j</groupId>
    <artifactId>log4j-api</artifactId>
    <version>2.25.3</version>
</dependency>

<!-- 强制使用 log4j-core 2.25.3 -->
<dependency>
    <groupId>org.apache.logging.log4j</groupId>
    <artifactId>log4j-core</artifactId>
    <version>2.25.3</version>
</dependency>

<!-- log4j-slf4j-impl -->
<dependency>
    <groupId>org.apache.logging.log4j</groupId>
    <artifactId>log4j-slf4j-impl</artifactId>
    <version>2.25.3</version>
</dependency>
```

### 3. 配置 SnakeYAML 版本（如果需要）

```xml
<dependency>
    <groupId>org.yaml</groupId>
    <artifactId>snakeyaml</artifactId>
    <version>1.25</version>
</dependency>
```

## ⚠️ 注意事项

### 1. 版本一致性
- 所有 log4j 相关依赖必须使用相同版本（2.25.3）
- 避免版本冲突

### 2. Spring Boot 兼容性
- 本依赖包已在 Spring Boot 2.2.2.RELEASE 上测试通过
- 其他版本请先测试

### 3. SnakeYAML 版本
- Spring Boot 2.2.2 需要 SnakeYAML 1.25
- 如果父 POM 使用了 2.0，需要在项目中覆盖

### 4. 必须排除 logback
- 使用 log4j2 时必须排除 Spring Boot 默认的 logback
- 否则会有冲突

## 📞 如需更多依赖

如果你的项目需要其他 Log4j 依赖（如 log4j-jul、log4j-web 等），可以：

### 方式1：修改导出脚本

编辑 `export-log4j-dependencies.sh`，取消注释可选依赖：

```bash
# 可选依赖（如果需要，可以取消注释）
declare -a OPTIONAL_DEPENDENCIES=(
    "org.apache.logging.log4j:log4j-jul:2.25.3"        # JUL 桥接
    "org.apache.logging.log4j:log4j-jcl:2.25.3"        # Commons Logging 桥接
    "org.apache.logging.log4j:log4j-web:2.25.3"        # Web 应用支持
    "org.apache.logging.log4j:log4j-1.2-api:2.25.3"    # Log4j 1.x 兼容
)
```

然后重新运行导出脚本。

### 方式2：手动下载

```bash
mvn dependency:get \
    -DgroupId=org.apache.logging.log4j \
    -DartifactId=log4j-web \
    -Dversion=2.25.3
```
