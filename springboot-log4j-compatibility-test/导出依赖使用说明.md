# Log4j 依赖导出工具使用说明

## 📋 功能说明

本工具用于将 **Log4j 2.25.3** 升级所需的依赖打包，以便导入到内网 Maven 仓库。

## 🎯 适用场景

- ✅ 内网环境无法访问外网 Maven 仓库
- ✅ 需要将 Spring Boot 2.2.2.RELEASE 的 log4j 升级到 2.25.3
- ✅ 需要批量导入依赖到 Nexus 私服

## 📦 包含的依赖

本工具会导出以下 Log4j 2.25.3 核心依赖：

| 依赖 | 版本 | 说明 |
|------|------|------|
| log4j-api | 2.25.3 | Log4j API |
| log4j-core | 2.25.3 | Log4j 核心实现 |
| log4j-slf4j-impl | 2.25.3 | SLF4J 到 Log4j 的桥接 |
| log4j-jul | 2.25.3 | JUL 到 Log4j 的桥接 |
| log4j-jcl | 2.25.3 | Commons Logging 到 Log4j 的桥接 |
| log4j-web | 2.25.3 | Web 应用支持 |
| log4j-1.2-api | 2.25.3 | Log4j 1.x 兼容层 |

## 🚀 使用步骤

### 第一步：执行导出脚本

```bash
cd /Users/huabin/workspace/playground/my-github/springboot/springboot-log4j-compatibility-test

# 执行导出脚本
./export-log4j-dependencies.sh
```

### 第二步：查看导出结果

脚本执行完成后，会生成：

```
log4j-dependencies-export/          # 导出目录
├── repository/                     # Maven 仓库结构
│   └── org/apache/logging/log4j/
│       ├── log4j-api/2.25.3/
│       ├── log4j-core/2.25.3/
│       ├── log4j-slf4j-impl/2.25.3/
│       └── ...
├── dependencies-list.txt           # 依赖清单
├── import-to-nexus.sh             # Nexus 导入脚本
├── settings.xml                   # Maven 配置
└── README.md                      # 使用说明

log4j-dependencies-YYYYMMDD_HHMMSS.zip  # ZIP 压缩包（约 2.0M）
```

### 第三步：传输到内网环境

```bash
# 将 ZIP 压缩包传输到内网服务器
scp log4j-dependencies-*.zip user@internal-server:/path/to/destination/
```

### 第四步：在内网环境解压

```bash
# SSH 到内网服务器
ssh user@internal-server

# 解压 ZIP 文件
unzip log4j-dependencies-*.zip
cd log4j-dependencies-export
```

### 第五步：配置 Nexus 信息

编辑 `import-to-nexus.sh`，修改以下配置：

```bash
# Nexus 仓库地址
NEXUS_URL="http://your-nexus-server:8081/repository/maven-releases/"

# Nexus 用户名
NEXUS_USER="admin"

# Nexus 密码
NEXUS_PASSWORD="admin123"
```

同时修改 `settings.xml` 中的认证信息：

```xml
<server>
    <id>nexus</id>
    <username>admin</username>
    <password>admin123</password>
</server>
```

### 第六步：执行导入

```bash
# 执行导入脚本
./import-to-nexus.sh
```

## 🔧 其他导入方式

### 方式1：复制到本地 Maven 仓库

```bash
# 直接复制到本地仓库
cp -r repository/* ~/.m2/repository/
```

### 方式2：使用本地文件仓库

在项目的 `pom.xml` 中添加：

```xml
<repositories>
    <repository>
        <id>local-log4j</id>
        <url>file://${project.basedir}/log4j-dependencies-export/repository</url>
    </repository>
</repositories>
```

### 方式3：手动上传到 Nexus

1. 登录 Nexus Web 界面
2. 选择目标仓库
3. 点击 "Upload" 按钮
4. 选择 jar 和 pom 文件上传

## ⚠️ 注意事项

### 1. 网络要求

- **外网环境**：执行 `export-log4j-dependencies.sh` 时需要能访问阿里云 Maven 镜像
- **内网环境**：执行 `import-to-nexus.sh` 时需要能访问内网 Nexus 服务器

### 2. Java 版本

- 导出脚本会自动切换到 JDK 1.8
- 如果系统没有 JDK 1.8，请先安装

### 3. Maven 要求

- 需要安装 Maven 3.x
- 确保 Maven 配置正确

### 4. Nexus 权限

- 确保 Nexus 用户有上传权限
- 确保目标仓库允许重复部署（如果需要）

## 🔍 验证导入

导入完成后，可以通过以下方式验证：

### 1. 在 Nexus 中查看

访问 Nexus Web 界面，搜索 `log4j-api`，应该能看到 2.25.3 版本。

### 2. 在项目中测试

```bash
# 在项目 pom.xml 中添加依赖
<dependency>
    <groupId>org.apache.logging.log4j</groupId>
    <artifactId>log4j-api</artifactId>
    <version>2.25.3</version>
</dependency>

# 执行 Maven 命令
mvn clean compile

# 查看依赖树
mvn dependency:tree | grep log4j
```

## 📝 常见问题

### Q1：导出脚本执行失败怎么办？

**A：** 检查以下几点：
1. 是否安装了 JDK 1.8
2. 是否安装了 Maven
3. 是否能访问阿里云 Maven 镜像
4. 查看错误日志定位问题

### Q2：导入到 Nexus 失败怎么办？

**A：** 检查以下几点：
1. Nexus URL 是否正确
2. 用户名密码是否正确
3. 用户是否有上传权限
4. 网络是否能访问 Nexus
5. 查看 Nexus 日志

### Q3：如何只导出部分依赖？

**A：** 编辑 `export-log4j-dependencies.sh`，修改 `DEPENDENCIES` 数组：

```bash
declare -a DEPENDENCIES=(
    "org.apache.logging.log4j:log4j-api:2.25.3"
    "org.apache.logging.log4j:log4j-core:2.25.3"
    # 注释掉不需要的依赖
    # "org.apache.logging.log4j:log4j-slf4j-impl:2.25.3"
)
```

### Q4：如何导出其他版本的 Log4j？

**A：** 编辑 `export-log4j-dependencies.sh`，修改版本号：

```bash
declare -a DEPENDENCIES=(
    "org.apache.logging.log4j:log4j-api:2.20.0"  # 修改版本号
    "org.apache.logging.log4j:log4j-core:2.20.0"
    # ...
)
```
