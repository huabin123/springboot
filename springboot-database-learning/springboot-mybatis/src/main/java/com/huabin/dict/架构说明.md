## MyBatis映射原理

### 1. SQL执行结果（ResultSet）
```
dict_id | dict_name | item_id | dict_id | item_name | item_value
--------|-----------|---------|---------|-----------|------------
   1    |   性别    |    1    |    1    |    男     |   male
   1    |   性别    |    2    |    1    |    女     |   female
   1    |   性别    |    3    |    1    |   未知    |  unknown
   2    |   状态    |    4    |    2    |   启用    |     1
   2    |   状态    |    5    |    2    |   禁用    |     0
```

### 2. MyBatis自动聚合（根据dict_id分组）
```
┌─────────────────────────────────────────────────────────┐
│ SysDictTypeVO (dict_id=1, dict_name="性别")             │
│ ├─ items[0]: SysDictItem(item_id=1, itemName="男")     │
│ ├─ items[1]: SysDictItem(item_id=2, itemName="女")     │
│ └─ items[2]: SysDictItem(item_id=3, itemName="未知")   │
└─────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────┐
│ SysDictTypeVO (dict_id=2, dict_name="状态")             │
│ ├─ items[0]: SysDictItem(item_id=4, itemName="启用")   │
│ └─ items[1]: SysDictItem(item_id=5, itemName="禁用")   │
└─────────────────────────────────────────────────────────┘
```

### 3. 最终返回List<SysDictTypeVO>
```java
[
  SysDictTypeVO(dictId=1, dictName="性别", items=[...]),
  SysDictTypeVO(dictId=2, dictName="状态", items=[...])
]
```

## 设计模式

### 1. 分层架构模式
- **Controller层**: 处理HTTP请求，参数校验
- **Service层**: 业务逻辑处理
- **Mapper层**: 数据访问
- **Entity/VO层**: 数据传输对象

### 2. DAO模式
- Mapper接口定义数据访问方法
- XML文件实现具体SQL逻辑
- MyBatis框架负责接口与XML的绑定

### 3. VO模式
- Entity: 数据库表的直接映射
- VO (View Object): 面向前端的数据传输对象
- 分离关注点，Entity变化不影响API

## 性能优化点

### 1. 单SQL查询
- 使用LEFT JOIN避免N+1查询问题
- 减少数据库往返次数

### 2. 连接池
- 使用HikariCP高性能连接池
- 配置合理的连接池参数

### 3. 索引优化
```sql
-- dict_pid索引：加速WHERE条件查询
INDEX idx_dict_pid (dict_pid)

-- dict_id索引：加速JOIN关联
INDEX idx_dict_id (dict_id)
```

### 4. 结果集映射
- MyBatis自动聚合，无需手动组装
- 减少内存开销和处理时间

## 总结

这个模块展示了如何使用MyBatis的高级特性（collection映射）来优雅地处理一对多关系查询。核心优势：

1. **单SQL实现**: 避免N+1查询问题
2. **自动映射**: MyBatis自动聚合结果集
3. **分层清晰**: 遵循MVC架构模式
4. **易于扩展**: 预留了多个扩展点
5. **性能优化**: 使用索引和连接池
