# Feign 无参构造方法问题 - 详细测试指南

## 测试环境准备

### 1. 确认环境

```bash
# 检查 JDK 版本
java -version
# 要求: JDK 1.8+

# 检查 Maven 版本
mvn -version
# 要求: Maven 3.6+

# 检查端口占用
```

```bash

# 编译 Provider Service
cd provider-service

cd ..
```

## 测试场景详解

### 场景 1: ❌ 复现问题（无参构造方法缺失）

#### 1.1 直接访问 Provider（本地调用 - 成功）

```bash
curl http://localhost:8081/api/users/problem/1
```

**预期输出**（成功）：
```json
{
  "id": 1,
  "username": "张三",
  "email": "zhangsan@example.com",
  "age": 25
}
```

**说明**：Provider 本地调用成功，因为不涉及 JSON 序列化/反序列化。

#### 1.2 通过 Feign 调用（远程调用 - 失败）

```bash
curl http://localhost:8082/test/problem/1
```

**预期输出**（失败）：
```json
{
  "success": false,
  "error": "HttpMessageNotReadableException",
  "message": "JSON parse error: Cannot construct instance of `com.huabin.consumer.dto.UserDTO` (no Creators, like default constructor, exist): cannot deserialize from Object value (no delegate- or property-based Creator)",
  "explanation": "因为 UserDTO 没有无参构造方法，JSON 反序列化失败"
}
```

#### 1.3 查看详细日志

**Provider 日志**（成功返回 JSON）：
```
Provider: 调用问题接口 /api/users/problem/1
Provider: 返回数据 UserDTO{id=1, username='张三', email='zhangsan@example.com', age=25}
```

**Consumer 日志**（反序列化失败）：
```
Consumer: 开始测试问题场景 - 调用 /api/users/problem/1
Consumer: ✅ 预期的错误发生了！错误信息: JSON parse error: Cannot construct instance...
```

#### 1.4 问题分析

```
Provider 端
    ↓
返回 JSON: {"id":1,"username":"张三","email":"zhangsan@example.com","age":25}
    ↓
Feign Client 接收
    ↓
Jackson 尝试反序列化
    ↓
调用 UserDTO 的无参构造方法  ← ❌ 失败！没有无参构造方法
    ↓
抛出异常: Cannot construct instance
```

---

### 场景 2: ✅ 问题修复（添加无参构造方法）

#### 2.1 直接访问 Provider（本地调用 - 成功）

```bash
curl http://localhost:8081/api/users/fixed/1
```

**预期输出**（成功）：
```json
{
  "id": 1,
  "username": "李四",
  "email": "lisi@example.com",
  "age": 30
}
```

#### 2.2 通过 Feign 调用（远程调用 - 成功）

```bash
curl http://localhost:8082/test/fixed/1
```

**预期输出**（成功）：
```json
{
  "success": true,
  "data": {
    "id": 1,
    "username": "李四",
    "email": "lisi@example.com",
    "age": 30
  },
  "message": "调用成功",
  "explanation": "因为 UserDTOFixed 有无参构造方法，JSON 反序列化成功"
}
```

#### 2.3 查看详细日志

**Provider 日志**：
```
Provider: 调用修复接口 /api/users/fixed/1
Provider: 返回数据 UserDTOFixed{id=1, username='李四', email='lisi@example.com', age=30}
```

**Consumer 日志**：
```
Consumer: 开始测试修复场景 - 调用 /api/users/fixed/1
Consumer: ✅ 调用成功！获取到用户: UserDTOFixed{id=1, username='李四', email='lisi@example.com', age=30}
```

#### 2.4 成功原因

```
Provider 端
    ↓
返回 JSON: {"id":1,"username":"李四","email":"lisi@example.com","age":30}
    ↓
Feign Client 接收
    ↓
Jackson 尝试反序列化
    ↓
调用 UserDTOFixed 的无参构造方法  ← ✅ 成功！
    ↓
通过反射设置属性值
    ↓
返回完整对象
```

```json
  "explanation": "因为 UserDTOLombok 使用 @NoArgsConstructor 生成了无参构造方法"
```

#### 3.3 Lombok 注解说明

```java

@NoArgsConstructor   // ✅ 生成无参构造方法
@AllArgsConstructor  // ✅ 生成全参构造方法
public class UserDTOLombok {
    private Long id;
    private String username;
    private String email;
    private Integer age;
}
```

## 常见错误排查

### 错误 1: 端口被占用

**错误信息**：
```
Web server failed to start. Port 8081 was already in use.
```

**解决方案**：
```bash
# 查找占用端口的进程
lsof -i :8081

# 杀死进程
kill -9 <PID>
```

### 错误 2: Maven 依赖下载失败

**错误信息**：
```
Could not resolve dependencies for project...
```

**解决方案**：
```bash
# 清理 Maven 缓存
mvn clean

# 强制更新依赖
mvn clean install -U

# 或删除本地仓库重新下载
rm -rf ~/.m2/repository/org/springframework/cloud
```

### 错误 3: Lombok 不生效

**错误信息**：
```
cannot find symbol: method builder()
```

**解决方案**：
1. 确认 IDE 安装了 Lombok 插件
2. 启用 Annotation Processing
3. 重新编译项目
